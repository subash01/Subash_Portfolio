CASE 01 MULTI-AGENT AI

The multi-agent system that gave employees 7 hours back a month

Enterprise HR & finance ops were death by a thousand forms. I orchestrated autonomous agents across expense, attendance, leave and approvals — with a human-in-the-loop ceiling.

−94%
Admin time
8h→30m
Per employee / wk
SOC2
Audit-ready
1Problem

Enterprise employees spent ~8 hours every week feeding the same data into expense, timesheet, leave and approval systems. The friction quietly taxed productivity across thousands of seats.

2Customer pain points
  • Repetitive manual entry across disconnected modules
  • Approval chains stalling on missing context
  • No trust in automation without an audit trail
3Discovery

Ran discovery across enterprise deployments to map where time actually leaked. The pattern: the work wasn't hard, it was repetitive and cross-module — perfect for orchestration, not a single bot.

4Research

Studied agent interaction patterns and failure modes for SOC2-compliant clients. Key insight: trust comes from fallback behaviour and an approve-to-act ceiling, not full autonomy.

5Solution

A multi-agent platform that orchestrates autonomous agents across expense, attendance and HR workflows. Each agent drafts and proposes; humans approve. I defined agent interaction patterns, conversational UX flows, fallback behaviours, edge-case handling, audit trails and rollback mechanisms for the approval system.

Architecture sketch — orchestration with human-in-the-loop
6PRD approach

Spec'd agent boundaries, escalation paths and rollback per workflow. Defined edge cases first — what happens when an agent is uncertain — so reliability was a requirement, not an afterthought.

7Metrics framework

Tracked time-to-complete per workflow, approval throughput, agent fallback rate and CSAT. Adoption gated on agents staying under a defined error threshold.

8Lessons learned

Trust is the real product in agentic AI. Customers didn't want magic — they wanted a system they could audit and override. The approve-to-act ceiling unlocked adoption that full autonomy never would have.

CASE 02 NO-CODE AI PLATFORM

Agent Studio: letting customers build their own AI agents

After shipping the Expense AI Agent (15→2 min), the real demand emerged: customers wanted to build agents themselves. So I architected a configurable, no-code agent platform.

15→2min
Expense submission
0
Eng dependency
No-code
Self-serve agents
1Problem

The Expense AI Agent proved agents could collapse a 15-minute chore to 2 minutes. But every new use case meant an engineering ticket — automation didn't scale.

2Customer pain points
  • Each new agent needed engineering time
  • Customers' workflows varied too much for one rigid bot
  • No way to customise without code
3Discovery

Across enterprise conversations, customers kept describing their own workflows. The opportunity wasn't more agents — it was a platform to let them compose agents themselves.

4Research

Benchmarked no-code builders and intent-recognition patterns. Defined the minimum primitives a non-technical admin needs to safely configure an agent: triggers, tools, guardrails, fallback.

5Solution

Agent Studio — a configurable, no-code platform where enterprise customers create, customise and deploy AI agents without engineering support. Paired with an AI Chat Assistant using NLP intent recognition for natural-language interaction across KEBS modules.

User flow — composing an agent without code
6Product decisions

Bet on configurability over more pre-built agents — a platform play. Constrained the builder to safe primitives so admins couldn't ship an agent that breaks compliance.

7Metrics framework

Measured agents created per customer, time-to-first-agent, and submission-time reduction per deployed agent (the original Expense Agent's 7.5× being the benchmark).

8Lessons learned

The biggest unlock in AI products is often not a smarter model — it's handing the configuration to the people who know the workflow. Platform > point solution once value is proven.

CASE 03 0→1 · REVENUE

An AI-powered ATS that turned 2 days of screening into 3 hours

A net-new 0→1 product. I researched, ideated and launched an enterprise Applicant Tracking System with AI candidate recommendations — and it opened a new revenue line.

2d→3h
Recruiter screening
+2
Enterprise customers
$0.25M
New revenue
1Problem

Recruiters drowned in manual résumé screening — up to two days per role — while KEBS lacked a hiring product to cross-sell into existing accounts.

2Customer pain points
  • Hours lost to manual shortlisting
  • Inconsistent screening quality
  • Hiring data siloed from the rest of operations
3Discovery

Validated demand directly with enterprise accounts before writing a line of spec — confirming both the pain and a willingness to pay for AI-assisted shortlisting.

4Research

Mapped the recruiter journey and where AI recommendations could rank candidates without removing recruiter judgement — assist, then let the human decide.

5Solution

A 0→1 enterprise ATS with AI-powered candidate recommendations that surface the strongest matches first, cutting screening from 2 days to 3 hours while keeping recruiters in control of the final call.

6PRD & build

Owned the full lifecycle: discovery → PRD → roadmap → launch, leading a team of 4 at 95% sprint completion. Built prototypes with a strong UX focus before committing engineering.

7Business impact

Acquired 2 new enterprise customers and contributed $0.25M in new revenue — turning a productivity feature into a standalone commercial line.

8Lessons learned

0→1 lives or dies on pre-build validation. Selling the outcome (3-hour screening) rather than the feature (an ATS) is what closed new logos.
